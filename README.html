Midnight Firearms

Multi-framework firearms access gate (Discord role + DB overrides) with an admin panel.
Author: MidnightMTB
Publisher: Ottawa Region Scripts
Frameworks: QBCore, Qbox, ESX, Standalone
Inventories: ox_inventory, qb/ps/lj/qs, others

What it does

  • Blocks firearm usage unless a player has a specific Discord role (ex: Alpha Civ).
  • Supports persistent overrides in MySQL: Allow or Deny.
  • Includes an admin panel (NUI) for viewing players and applying overrides.
  • Framework-agnostic enforcement (works even if your inventory is different).

Dependencies

  • ox_lib
  • oxmysql

Recommended

  • Keep your server artifacts updated for stable HTTPS requests to Discord.
  • Use a dedicated Discord bot just for server automation/security.

Install (Quick)

  1. Drop the resource folder into your server resources.
  2. Import SQL: DB.sql
  3. Add ensures in server.cfg (order matters).
  4. Configure config.lua (Guild ID, Role ID, Bot Token).
  5. Give yourself ACE permission for the panel/admin actions.

server.cfg ensure order

ensure oxmysql
ensure ox_lib
ensure midnight_firearms

ACE permission

add_ace group.admin midnight_firearms.admin allow

Panel command: /mf_panel (default). Keybind: F7 (default).

Discord Bot Setup

This script checks guild member roles through the Discord API. That means you need a bot token.

1) Create an application & bot

  • Go to the Discord Developer Portal.
  • Create a New Application.
  • Open the Bot section and click Add Bot.

2) Copy the Bot Token

  • In the Bot page, click Reset Token (if needed), then copy the token.
  • Paste into config.lua:
Config.BotToken = 'YOUR_BOT_TOKEN_HERE'

3) Invite the bot to your server

  • In the Developer Portal, go to OAuth2 → URL Generator.
  • Select scope: bot
  • Select permissions: minimal is fine, but it must be in the guild.
  • Open the generated URL and invite the bot to your Discord server.

4) Get Guild ID and Role ID

  • Enable Developer Mode in Discord (Settings → Advanced).
  • Right-click your server → Copy ID → paste into Config.GuildId.
  • Right-click the role → Copy ID → paste into Config.AllowedRoleId.
Config.GuildId = 'YOUR_GUILD_ID'
Config.AllowedRoleId = 'YOUR_ROLE_ID'

Common Errors

401 Unauthorized Bot token is invalid/misformatted.

  • Re-copy the bot token from the Developer Portal.
  • Make sure you pasted the bot token, not the client secret.
  • Restart the resource or server after changes.

403 Forbidden Bot lacks access or you’re hitting a permission/intents issue.

  • Ensure the bot is invited to the correct guild.
  • Verify you’re using the correct guild ID and role ID.

429 Rate Limited Too many role checks too fast.

  • Increase Config.RefreshSeconds.
  • Avoid spamming refresh on many players at once.
